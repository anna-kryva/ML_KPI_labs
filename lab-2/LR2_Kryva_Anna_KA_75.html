Варіант 5, КА-75¶

Побудувати моделi класифiкацiї на основi методу дерев рiшень, викори стовуючи sklearn.tree.DecisionTreeClassifier з рiзними значеннями гiперпараметрiв:
• max_depth – максимальна глибина дерева,
• min_samples_split – мiнiмальна кiлькiсть прикладiв, якi мають бути у вузлi, перш нiж його можна буде розщепити,
• min_samples_leaf – мiнiмальна кiлькiсть прикладiв у листовому вузлi,
• max_leaf_nodes – максимальна кiлькiсть листових вузлiв,
• max_features – максимальна кiлькiсть ознак, якi оцiнюються при розщепленнi кожного вузла.

Побудувати границi рiшень графiчно для кожної з моделей.
Настроїти гiперпараметри дерева за допомогою перехресної перевiрки.

Початковi данi:
(а) sklearn.datasets.load_digits
(б) sklearn.datasets.make_moons

import numpy as np

def get_grid(data):
    x_min, x_max = data[:, 0].min() - 1, data[:, 0].max() + 1
    y_min, y_max = data[:, 1].min() - 1, data[:, 1].max() + 1
    return np.meshgrid(np.arange(x_min, x_max, 0.01), np.arange(y_min, y_max, 0.01))

xx, yy = get_grid(X_moons_train)
predicted = decisionTree_moons.predict(np.c_[xx.ravel(), yy.ravel()]).reshape(xx.shape)

10. Зробити висновки про якiсть роботи моделей на дослiджених даних. Для кожної навчальної вибiрки вибрати найкращу модель.¶

Дерево рішень добре працює для датасету з невеликою кількістю класів, таких як make_moons. Високу точність для даних з великою кількістю класів важко досягти.
Для даних load_digits я виберу модель з найденими гіпермараметрами за допомогою решітчатого пошуку.
Для даних make_moons обрати кращу модель важко, оскільки точність майже співпадає. В даному випадку виберу з гіпермараметрами, які були знайдені решітчатим пошуком, оскільки значення в них менші, що говорить про більш швидку роботу.

Висновки¶

У даній роботі були побудовані моделі за допомогою дерева рішень бібліотеки sclearn. Модель з меншою кількістю класів дала гарний результат. Модель з великою кількістю класів дала не високу точність прогнозу. Це пов'язано з тим, що потрібно багато ресурсів для її побудови та знаходження найкращих параметрів.
Точність моделі було оцінено за допомогою метрик precision, recall, f1-score та confusion matrix. Були побудовоані графіки, за допомогою ких можна розглянути, які ознаки брала модель для класифікації. За допомогою решітчатого пошуку було підібрано найкращі гупермараметри моделі.
